1. 프로그램 주제와 목적
내 프로그램의 주제: 만 나이 기반 맞춤형 상비약 추천 및 복용 안내 프로그램
이 프로그램을 만든 이유: 일상생활에서 갑작스러운 통증이나 증상이 발생했을 때, 집안에 구비된 상비약을 안전하고 올바르게 선택할 수 있도록 돕고자 만들었습니다. 특히 성인과 어린이의 복용 기준이 다른 약물의 특성을 고려하여, 만 나이 제한에 따른 약물 오남용 위험을 사전에 차단하고 안전하게 복용할 수 있는 환경을 제공하기 위함입니다.
2. 문제 분해 및 모델링
입력
사용자가 현재 겪고 있는 증상 (문자열: 두통, 소화불량, 감기, 알레르기 중 선택)
사용자의 만 나이 (정수형 숫자)
처리
2차원 리스트(medicine_data)를 반복문으로 한 줄씩 돌며 사용자가 입력한 증상과 일치하는 약 데이터가 있는지 탐색합니다.
만약 일치하는 약을 찾았는데, 사용자의 나이가 만 12세 미만이고 해당 약이 어린이 복용 불가("N") 데이터라면 "어린이불가"라는 특별한 문자열 신호를 생성합니다.
나이 제한에 걸리지 않고 정상적인 약을 찾았다면 해당 약의 정보 리스트 전체를 반환하고, 일치하는 증상이 없다면 None을 생성합니다.
출력
정상 추천인 경우: 추천 약의 이름과 올바른 복용 방법을 가독성 있게 출력합니다.
어린이 복용 제한인 경우: 어린이 복용 불가 및 의사·약사 상의 권고 경고 메시지를 출력합니다.
데이터가 없는 경우: 입력한 증상에 맞는 약을 찾을 수 없다는 예외 안내 메시지를 출력합니다.

2차원 리스트 이름: medicine_data
한 행의 의미: 프로그램에 등록된 개별 상비약 1개의 전체 정보 (약 하나당 4가지 속성을 가짐)
각 열의 의미:
열 번호
의미
실제 예시

약의 상품 이름 (문자열)

안전하고 올바른 복용 방법 안내 (문자열)

이 2차원 리스트를 사용한 이유: 약 이름, 증상, 복용법, 연령 제한이라는 4가지 서로 다른 성격의 데이터들을 하나의 묶음(행)으로 결합하고, 여러 개의 약 데이터를 인덱스(열 번호)를 통해 체계적이고 효율적으로 관리하기 위해 사용했습니다.

가장 중요한 함수 하나를 골라 자세히 설명하세요.
함수 이름: search_medicine(data_list, user_symptom, user_age)
설명: 프로그램의 핵심 알고리즘(데이터 처리)을 전담하는 함수입니다. 메인 함수로부터 약 데이터 전체와 사용자의 입력값들을 전달받아 for 반복문으로 2차원 리스트를 순회합니다. 사용자의 증상과 med[1]이 일치하는 행을 찾으면, 다시 if 조건문으로 만 12세 미만 여부와 med[3] == "N" 조건을 검사하여 오남용 위험을 판정합니다. 이 함수는 화면 출력을 직접 하지 않고 오직 데이터 판정 후 결과 신호만 리턴하는 구조로 설계되어 프로그램의 가독성을 높여줍니다.
5. 조건문과 반복문 활용
조건문 사용 부분
어떤 조건을 검사했는가?
if med[1] == user_symptom: 현재 탐색 중인 약 데이터의 증상 카테고리가 사용자가 입력한 현재 증상과 일치하는지 검사했습니다.
if user_age < 12 and med[3] == "N": 사용자가 만 12세 미만 어린이인지 확인하고, 동시에 해당 약물이 어린이 복용 불가능한 약물인지 다중 조건을 검사했습니다.
if result == "어린이불가", elif result is not None, else: 판정 결과 신호에 따라 어떤 안내 메시지를 출력할지 분기 조건을 검사했습니다.
반복문 사용 부분
무엇을 반복 처리했는가?
for med in data_list: 코드를 사용하여 2차원 리스트 medicine_data 안에 저장된 모든 약 데이터 행들을 첫 번째 행부터 마지막 행까지 차례대로 순회하며 반복 처리했습니다.
조건문과 반복문이 함께 작동한 부분
반복문(for)이 약 데이터를 한 줄씩 꺼내오는 과정 내부에서, 조건문(if)이 작동하여 "증상이 일치하는가?"를 먼저 검사하고, 일치할 경우 그 안에서 다시 중첩 조건문(if)이 "어린이 제한에 걸리는가?"를 판단하여 리스트 전체를 효율적으로 필터링하도록 함께 작동했습니다.

수정 방법: 화면 출력의 역할을 분리하여 모든 출력은 메인 흐름 및 출력 전담 함수가 처리하도록 규칙을 바꿨습니다. 검색 함수 안의 print를 제거하고, 제한에 걸릴 시 "어린이불가"라는 특별한 문자열 신호를 반환(return)하게 한 뒤 main 함수에서 if-elif-else문으로 분리하여 출력하도록 구조를 수정하여 버그를 해결했습니다.
8. Gemini 활용 내용 요약
Gemini에게 도움받은 부분:
고등학교 정보 수업 범위에 맞는 2차원 리스트 구조 설계와 핵심 함수(검색 및 제어)의 뼈대 코드를 제안받았습니다.
조건문 중첩 시 발생했던 중복 출력 오류의 정확한 원인 분석과 출력 기능을 일원화하는 해결 아이디어를 도움받았습니다.
Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:
Gemini가 처음에 어린이 제한 신호로 영문 문자열인 "CHILD_RESTRICTED"를 반환하라고 제안했으나, 코드인터뷰 시 제가 더 직관적으로 설명하고 이해하기 쉽도록 직관적인 한국어 신호인 "어린이불가"로 수정하여 사용했습니다.
최종 코드에서 내가 직접 판단하거나 수정한 부분:
프로그램의 가독성과 함수 분리 조건을 더 깊이 있게 만족시키기 위해, main 함수 안에 있던 정상 출력 부분을 print_medicine_info(medicine)라는 3번째 독립된 함수로 직접 추가 분리하여 구현했습니다.
9. 한계와 개선 방안
현재 프로그램의 한계:
현재 프로그램은 사용자가 나이를 입력할 때 숫자가 아닌 글자(예: '스무살')를 입력하면 숫자로 변환하는 int() 과정에서 프로그램이 강제로 종료되는 한계가 있습니다.
동일한 증상에 대해 오직 한 가지의 약만 추천할 수 있는 고정된 탐색 구조를 가지고 있습니다.
다시 개선한다면 추가하고 싶은 기능:
try - except 예외 처리 문법을 학습하여 사용자가 나이에 잘못된 형식을 입력해도 정중하게 재입력을 요구하는 방어 코드를 추가하고 싶습니다.
동일한 증상(예: 두통)에 여러 개의 약 데이터가 있을 때, 조건에 맞는 모든 약을 리스트로 묶어 한꺼번에 추천해 주는 다중 검색 기능을 추가하고 싶습니다.
10. 코드인터뷰 준비
다음 질문에 답할 수 있도록 준비하세요. (스스로 연습해 보세요!)
내 프로그램의 입력-처리-출력은 무엇인가?
위 2번 항목에 적힌 대로 증상/나이를 받아 리스트 탐색 후 연령을 판정하고 결과를 화면에 띄우는 흐름입니다.
2차원 리스트에서 한 행과 각 열은 무엇을 의미하는가?
한 행은 약 1개의 정보이고, 0번 열은 이름, 1번 열은 증상, 2번 열은 복용법, 3번 열은 어린이 가능 여부입니다.
가장 중요한 함수는 무엇이고 어떤 역할을 하는가?
search_medicine 함수이며, 반복문과 조건문을 활용해 데이터를 탐색하고 안전성 신호를 판정하여 반환합니다.
조건문과 반복문은 어디에서 사용되었는가?
search_medicine 함수 내부에서 리스트 전체를 순회할 때 for 반복문을 썼고, 증상 일치 및 나이 제한을 검사할 때 if 조건문을 썼습니다.
Gemini가 도와준 부분과 내가 직접 수정한 부분은 무엇인가?
중복 출력 버그의 원인과 신호 반환 아이디어를 Gemini에게 도움받았고, 영문 신호를 한국어로 고치고 출력 전담 함수(print_medicine_info)를 따로 추가 정의한 것은 제가 직접 수정하고 판단한 부분입니다.
