# Gemini 활용 기록

> 이번 수행평가에서는 학교 계정 Gemini 사용을 허용합니다.  
> 단, Gemini가 만든 코드를 이해하지 못한 채 그대로 제출하면 안 됩니다.  
> 실제 개인정보, 친구 이름, 성적, 전화번호, 주소 등은 절대 입력하지 않습니다.

## 1차 Gemini 활용

- 사용 날짜:6/10
- 사용 목적: 학교 정보 과목 파이썬 미니 프로젝트 수행평가를 위해, 필수 조건(2차원 리스트, 함수 분리, 제어구조)을 만족하는 프로그램을 기획하고 개발하는 과정에서 아이디어를 구체화하고 발생한 코드 버그를 해결하기 위함.



```text
여기에 내가 Gemini에게 입력한 질문을 붙여넣기
질문 1 (주제 기획 단계): "약 판정기" (이 아이디어로 프로젝트를 하고 싶다고 전함)

질문 2 (코드 구현 단계): (빈칸 코드를 직접 채운 뒤 코드 조각을 보여주며) "잘했지"

질문 3 (디버깅 단계): (중복 출력 버그를 고치는 구조 변경 힌트를 들은 후 헷갈려서) "모르겠어"

- Gemini가 제안한 내용 요약:주제 및 데이터 구조 제안: 학생 수준에 맞게 "나에게 맞는 맞춤형 상비약/영양제 추천 및 복용 안내기"로 주제를 좁혀주고, 약 이름·증상·복용량·어린이 가능 여부를 담은 2차원 리스트 구조와 핵심 함수 뼈대를 제안함.

버그 원인 분석 및 해결책 제안: 검색 함수와 메인 함수 양쪽에서 모두 print()를 사용해 메시지가 중복 출력되는 버그의 원인을 짚어줌. 화면 출력은 main() 함수가 전담하도록 역할을 분리하고, 함수가 "어린이불가"라는 특별한 문자열 신호(반환값)를 보내 메인 함수에서 if-elif-else문으로 제어하라는 해결책을 제안함.
- 내가 반영한 부분:2차원 리스트 데이터 정의: 제안받은 medicine_data 리스트 구조를 그대로 프로그램에 반영함.

함수 빈칸 구현: search_medicine 함수 내부에서 조건에 맞는 약 정보를 found_medicine = med로 저장하는 알고리즘과, main 함수에서 약을 찾지 못했을 때의 예외 메시지 출력 코드를 직접 작성하여 반영함.

출력 구조 일원화 및 버그 수정: Gemini의 조언대로 검색 함수 안의 print문을 지우고, 어린이가 복용 불가능할 때 "어린이불가" 신호를 리턴하도록 코드를 수정함. main() 함수에서 result == "어린이불가" 조건을 추가하여 중복 출력 버그를 완벽히 해결함.
- 내가 수정하거나 사용하지 않은 부분:첫 번째 버그 수정 제안: Gemini가 처음 제안했던 "어린이 제한 시 return None 대신 "CHILD_RESTRICTED"라는 영문 신호를 보내고 main 함수 구조를 세분화하는 방법"은 코드가 복잡하고 직관적으로 이해하기 어려워 적용하지 않음. 대신 "출력 기능은 main 함수가 전담하고 한국어 신호("어린이불가")를 사용하자"는 두 번째 제안이 더 명확하여 이를 선택해 코드를 수정함.

## 3차 Gemini 활용

- 사용 날짜:6/16
- 사용 목적:학교 정보 과목 파이썬 미니 프로젝트 수행평가의 최종 단계인 '최종 수행평가 보고서(report.md)'를 학교 공식 양식에 맞춰 성실하게 작성하고, 프로젝트의 성과와 한계점을 객관적으로 정리하기 위함.
- 내가 입력한 프롬프트:"최종 보고서 써줘" (최종 보고서 작성을 위한 기본 가이드 및 내용 정리 요청)


```

- Gemini가 제안한 내용 요약:학생이 제시한 10가지 공식 보고서 항목(주제, 문제 모델링, 2차원 리스트 구조, 함수 설명, 제어구조 활용, 실행 결과 분석, 오류 수정 경험, Gemini 활용 요약, 한계와 개선 방안, 코드인터뷰 준비)에 맞추어, 우리가 완성한 '함수 3개짜리 약 판정기 프로그램'의 핵심 논리와 구조를 정확하게 매칭하여 서술해 줌.

## AI 활용 성찰

Gemini가 도움이 된 점:아이디어 구체화 및 구조 설계: '약 판정기'라는 추상적인 아이디어를 고등학교 정보 수업 범위에 맞게 '만 나이 기반 맞춤형 상비약 추천 프로그램'으로 구체화해 주었고, 4가지 속성을 가진 2차원 리스트(medicine_data) 구조의 기초 뼈대를 설계하는 데 큰 도움이 되었습니다.

논리적 버그 원인 분석: 조건문 검사 과정에서 두 가지 안내 메시지가 동시에 중복 출력되는 버그가 발생했을 때, 여러 함수에서 동시에 print() 출력을 수행하여 발생한 흐름 제어의 문제라는 점을 명확하게 짚어주어 문제의 원인을 쉽게 파악할 수 있었습니다.

출력 일원화 아이디어 제공: 화면 출력은 main() 함수 측이 전담하도록 하고, 검색 함수는 결과에 따라 특정한 '반환값(신호)'만 넘겨주어 메인 흐름에서 조건문으로 깔끔하게 제어하라는 논리적 해결 방향을 제안해 주었습니다.


Gemini의 제안을 그대로 쓰지 않고 내가 판단하거나 수정한 점:직관적인 한국어 신호로 변경: Gemini는 처음에 어린이 제한 판정 시 영문 문자열 신호인 "CHILD_RESTRICTED"를 반환하라고 제안했습니다. 하지만 제가 코드를 더 직관적으로 이해하고 최종 코드인터뷰에서 선생님께 명확하게 설명할 수 있도록, 제 판단 하에 더 직관적이고 알기 쉬운 한국어 문자열 신호인 "어린이불가"로 변경하여 적용했습니다.

기능별 함수 추가 분리 구현: Gemini가 제안한 초기 코드에서는 main() 함수가 입력과 제어, 정상 약 정보 출력까지 모두 담당하고 있었습니다. 저는 함수 분리 조건을 더 깊이 있게 만족하고 코드의 가독성을 높이기 위해, 정상 약 정보를 화면에 출력하는 부분을 print_medicine_info(medicine)라는 3번째 독립된 함수로 직접 정의하고 분리하여 호출하도록 코드를 수정했습니다.


다음에 AI를 사용할 때 주의해야 할 점:AI의 답변을 비판적으로 수용하기: AI가 제안하는 코드나 영문 변수/신호명이 항상 내 수준이나 프로그램에 가장 베스트가 아닐 수 있음을 배웠습니다. 무작정 복사해서 붙여넣기보다는 '내가 직접 남에게 설명할 수 있는 코드인가?'를 먼저 판단하고, 필요에 따라 내 스타일과 수준에 맞게 다듬는 과정이 꼭 필요하다는 것을 깨달았습니다.

개인정보 및 민감 정보 보호: 건강이나 약물과 관련된 주제를 다룰 때 실제 주변 친구들의 이름, 나이, 실제 앓고 있는 질환 등 민감한 개인정보를 데이터에 입력하지 않도록 주의해야 하며, 항상 가상의 데이터를 설계하여 다루어야 함을 배웠습니다.

질문의 구체성 높이기: 코드가 막혔을 때 단순히 "모르겠어"라고 묻기보다는, 어느 부분이 헷갈리는지 내 생각을 구체적으로 정리해서 질문할 때 AI로부터 더 명확하고 유용한 힌트를 얻을 수 있다는 점을 알게 되었습니다.
